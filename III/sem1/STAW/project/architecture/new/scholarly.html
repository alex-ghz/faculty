Cerintele Aplicatiei

  • Gestionarea in timp real a fluxurilor de activitati din cadrul unui lant de furnizori de energie, baterii si acumulatori speciali pentru drone, biciclete si autoturisme electrice.
  • Posibilitatea de a monitoriza de la distanta stocul de baterii.
  • Plata energiei livrate.
  • Notificarea clientilor fideli asupra schimbarilor survenite (modificari de pret, indisponibilitatea anumitor produse, oferte speciale, etc.) utilizand fluxuri RSS.
  • Localizarea statiilor de furnizare a energiei electrice pe harta folosind servicii Web cartografice.
  • Rapoarte si vizualizari atractive privitoare la stocurile de baterii si a situatiei vanzarilor per sortiment, perioada de timp sau in functie de client.
  • Arhitectura modulara a aplicatiei.
  • Implementarea extensiilor pentru functionalitati suplimentare.

Arhitectura Aplicatiei

  • REST (Representational State Transfer) Architecture
  • MVC (Model - View - Controller) Pattern

Tehnologii Utilizate in Cadrul Aplicatiei

  • Client Interface - Embedded JavaScript Templates, Less.js & Client-Side JavaScript
  • Business Logic - Express.js
  • APIs - Express.js
  • Database - MongoDB

Interactiunea Client-Server

  • Client Interface - Dynamic Content w/ EJS, Client-Side JS and AJAX.
  • Business Logic - Doesn't interact directly with Data from Database; Deliver Local Data and Database Data from API to Client.
  • API - Directly access and deliver Data from Database.
  • Client -> GET Req -> Server -> EJS Res -> Client -> AJAX Req & Sync w/ Promise -> Server -> Req Extern API -> JSON Res -> Server -> JSON Res -> Client w\ Promise -> Update Content.

Structura Codului

  • Client Interface - Client-Side JavaScript w\ Event Listeners and AJAX Requests.
  • Business Logic - Express.js Routes, Script Manager & API Requests. Get JSON Data from API, Send JSON Data to Client.
  • API - Express.js Routes and Models; Loads all Models when API Starts; Query Database on Req and Send Data as JSON.

Client Interface

  • Index w\ Map
  • Login
  • Register
  • My Account
  • Shop
  • EJS, Client-Side JavaScript w\ Event Listeners and AJAX for Dynamic Content and Notifications.
  • Get and Load Data just when End-User needs it.

Business Logic

  • Routes: Index, Admin, My Account, Notifications, Shop.
  • Script Manager: Login, Notifications, Shop, Page Content for Admin and User.
  • Express.js; Get Req from Client; Req API for Data; Get Data from API; Modify EJS and render it to Client.

API

  • API over ElecTrik MongoDB Database.
  • Models Objects for Data: Admin, Filter, Notifications, Products, User.
  • Loads all Models when at start-up.
  • Read and Write Data into Database; JSON Res to Business Logic's Req.
